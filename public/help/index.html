Idris online map tools

Introduction

With our online tools, you can edit geographic data and create maps directly from your browser. Without downloading mapping software on your machine. It even works on your mobile phone.

Everything happens in your browser. No data is being sent to our servers. You can work with your sensitive data without worrying about it traveling over the net.

Get started

Do you already have geographic data?

Get geographic data

Open street map

OSM is the wikipedia of geodata. There are different ways to get hold of it. Have a look at their wiki. Or try some of the options below.

Overpass turbo

Overpass turbo is an online tool to download data from OSM

Mapzen

Mapzen has some custom extracts of OSM data such as Administrative areas by country or Extracts for metropolitan areas.

When using data from OSM, or any other highly detailed geodata, do not forget to simplify it first. If you are making a world map, you do not need every single detail of the norwegian fjords for example. Mapshaper is an online tool (not related to Idris maps) that lets you do that.

Natural earth

Natural Earth Data has a wide range of downloadable worldwide geodata.

Idris geodata

We have a few datasets to get started on github. The collection is small at the moment but we will be adding datasets in the future.

Draw your own

You can create your own data by drawing it yourself with this tool and save it as GeoJSON

Geographic data formats

Geographic data comes in many formats. For our tools we use the GeoJSON format but we provide converters for the most common ones.

Convert GPX files

Go to the tool here and drop the file in the designated area.

Convert data from databases and CSV files

If your geodata is in a database, chances are you can extract it as GeoJSON directly. In PostGIS, you can use "ST_AsGeoJSON()" for example.

If not, export it as a WKT column in a CSV file. Go to the tool here and drop the CSV file in the designated area.

For geodata, the CSV converter takes either a WKT column or two columns with latitude and longitude for points.

If you have data on a spreadsheet, you can paste it here.

For those working with databases, we also have a tool that creates an SQL script (geodata is in WKT) from a GeoJSON file.

Edit a GeoJSON file

Select features from your GeoJSON file by bounding box, by property or by clicking on a map.

Edit the properties of your GeoJSON file with this tool. It lets you remove properties or add some from a CSV file by joining a property with a column in the file.

Make a map

Open the map tool and drop a GeoJSON file

Style layers

Polygons

fill: is the color of the polygons. It takes any valid HTML color.

fill-opacity: is the opacity the polygons. It takes a value between 0 and 1.

stroke: is the color of the outlines. It takes any valid HTML color

stroke-opacity: is the opacity the outlines. It takes a value between 0 and 1.

stroke-width: is the width of the outlines. It takes any numeric value.

Lines

stroke: is the color of the lines. It takes any valid HTML color

stroke-opacity: is the opacity the lines. It takes a value between 0 and 1.

stroke-width: is the width of the lines. It takes any numeric value.

Points

Points can be presented either as circles or as labels. Circles are used by default, change to labels by clicking the "Show as labels" button.

Points as circles

r: is the radius of the circles. It takes any numeric value.

fill: is the color of the circles. It takes any valid HTML color.

fill-opacity: is the opacity the circles. It takes a value between 0 and 1.

stroke: is the color of the outines. It takes any valid HTML color

stroke-opacity: is the opacity the outlines. It takes a value between 0 and 1.

stroke-width: is the width of the outlines. It takes any numeric value.

Points as labels

fill: is the color of the labels. It takes any valid HTML color.

font-family: is the typography. It takes any font recognised by your browser.

font-size: is the size of the labels. It takes any numeric value.

text-anchor: is the position of the text. It can be "start", "middle" or "end".

Style by property

To style individual features according to properties, use the "Style by property" button. Any of the styling possibilities above can be used (fill, stroke, r, font-size...). For properties with numeric values, you can choose to use rules. For example if you have a property "population" on polygons you can set the "fill" to orange for values above 10000 and red for values above 20000. For non-numeric values, you can enter the style by hand.

Save

When you are satisfied with your map you can save it by clicking "Download map".

By default the map will try to fit on your screen. So the canvas size is defined by the size of your screen and the projection is defined in order to fit the first layer. You can change both by clicking "Set map size" and "Set bounding box".

The map will be downloaded as a SVG file. If you wish to convert it to PNG, use this tool.
